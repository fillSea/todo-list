# 日历页周期任务重复显示修复方案

## 目标
修复 `miniprogram/pages/calendar/` 中“查看进行中的周期任务时，同一系列任务同时出现在进行中和已完成区域”的问题，使周期任务在选中日期任务列表中只按预期显示一次，并保持现有周期规则、云函数协议和首页行为一致。

## 问题结论
问题主要发生在 `miniprogram/pages/calendar/calendar.js` 的 `loadTasksForSelectedDate()`。

当前逻辑：
- 直接用 `isTaskRepeatingOnDate(task, selectedDate)` 过滤所有任务。
- 对于父周期任务，只要符合重复规则，就会被认为“存在于该日期”。
- 如果该系列在该日期已经有真实实例任务（`isPeriodicInstance: true`）且状态为未完成，父任务仍可能因为重复规则命中同一天。
- 之后页面再按 `status` 分成 `inProgressTasks` / `completedTasks`，于是就会出现：
  - 一个真实实例在“进行中”
  - 同系列父任务在“已完成”
- 用户感知为“同一个周期任务出现两条，一条进行中，一条已完成”。

这属于前端展示层的系列去重缺失，不是云函数返回格式错误。

## 已确认的根因
### 1. 选中日期列表没有做“按周期系列 + 按日期”的去重
位置：`calendar.js` 的 `loadTasksForSelectedDate()`

现状：
- 所有命中 `isTaskRepeatingOnDate()` 的任务都会进入 `selectedDateTasks`。
- 没有区分：
  - “该日期的真实任务实例”
  - “仅用于推导重复存在性的父任务投影”

影响：
- 只要一个系列里既有完成过的父任务，又有当天未完成实例，就会跨状态区块重复显示。

### 2. 父任务的“投影显示”优先级没有低于真实实例
现状：
- 父任务在未来/后续重复日期只是“规则投影”，并不一定代表用户真正应操作的那一条任务。
- 真实实例一旦存在，UI 应优先展示真实实例，而不是父任务和实例同时展示。

影响：
- 父任务污染当天展示结果。
- 完成状态和操作入口都可能落到错误对象上。

## 设计原则
1. 不修改云函数接口和存储结构。
2. 问题优先在日历页展示层修复。
3. 保持现有 `isTaskRepeatingOnDate()` 的用途：
   - 继续用于日历徽章与周期存在性判断。
4. 选中日期任务列表使用更严格的“展示候选集”规则：
   - 真实实例优先
   - 同系列只显示一条对应当天的任务
5. 与首页 / 清单详情页的“周期系列去重”思路保持一致，但保留日历页按日期展示的特点。

## 推荐方案
### 方案：在 `loadTasksForSelectedDate()` 前增加“选中日期展示任务构建层”
新增一个页面内辅助方法，例如：
- `getDisplayTasksForSelectedDate(selectedDate, tasks)`

职责：
1. 先找出所有“命中该日期”的任务候选。
2. 按 `getTaskSeriesGroupId(task)` 对周期任务分组。
3. 对每个周期系列按以下优先级选出当日唯一展示对象。

建议优先级：
1. 该日期的真实任务（`getUTC8DateString(task.dueDate) === selectedDate`）
2. 如果没有真实任务，再退回父任务的重复投影（`!task.isPeriodicInstance && isTaskRepeatingOnDate(task, selectedDate)`）

进一步规则：
- 非周期任务：保持原样
- 周期任务：每个系列最多保留一条用于列表展示
- 一旦该系列在该日期存在真实任务，就完全屏蔽该系列的父任务投影

这样可以覆盖最典型重复场景：
- 父任务：已完成
- 当天实例：未完成
- 最终只显示当天实例，不再额外显示父任务

## 实施细化
### 第一部分：抽出选中日期候选构建逻辑
在 `calendar.js` 中新增辅助方法，建议拆成以下两个小步骤。

1. `getSelectedDateTaskCandidates(selectedDate, tasks)`
- 返回所有理论上命中该日期的任务候选
- 可继续使用 `isTaskRepeatingOnDate()` 做基础筛选

2. `dedupeSelectedDatePeriodicTasks(taskCandidates, selectedDate)`
- 将候选任务分成：
  - 非周期任务
  - 周期任务
- 周期任务按系列分组后做展示去重

### 第二部分：系列去重规则
对于同一 `seriesId = getTaskSeriesGroupId(task)`：

1. 先取该系列中 `dueDate` 精确等于 `selectedDate` 的任务
- 这是“真实当天任务”
- 如果存在：
  - 只保留这些真实任务中的一个展示对象
  - 丢弃该系列的父任务投影

2. 如果没有真实当天任务
- 才允许用父任务投影补位
- 只保留一条父任务投影

### 第三部分：真实当天任务冲突时的兜底选择
如果同一系列、同一天出现多个真实任务（理论上不应常见，但可以防脏数据）：

推荐选择顺序：
1. `status === 0` 的未完成任务优先
2. 若状态相同，优先 `isPeriodicInstance === true`
3. 再按 `updatedAt` 或 `createdAt` 较新者优先
4. 最后回退到数组首项

说明：
- 正常数据里，这个兜底不应频繁触发。
- 加上它可以避免历史脏数据继续在 UI 中放大问题。

## 代码落点
### 主要修改文件
- `miniprogram/pages/calendar/calendar.js`

### 重点改动位置
1. `loadTasksForSelectedDate()`
- 不再直接把 `tasks.filter(isTaskRepeatingOnDate)` 的结果作为最终展示集
- 改为：
  - 先构建候选
  - 再对周期任务按系列去重
  - 再进行分类筛选、逾期派生和 `inProgress/overdue/completed` 分组

2. 复用已有工具
- `getTaskSeriesGroupId`
- `getUTC8DateString`
- `isTaskRepeatingOnDate`

### 不建议改动的地方
- 云函数 `taskFunctions`
- `isTaskRepeatingOnDate()` 的核心“是否命中日期”能力
- 日历 `periodicCountMap` 的计算方式

原因：
- 当前重复问题出在“列表展示集合”的构造，而不是周期存在性判断本身。
- 若直接改 `isTaskRepeatingOnDate()`，可能误伤徽章、统计和未来日期投影能力。

## 风险与注意点
### 1. 不要把所有已完成父任务都全局隐藏
错误做法：
- 简单地过滤 `status === 1 && repeatType > 0 && !isPeriodicInstance`

风险：
- 会导致在父任务原始日期上，本应显示的已完成周期任务消失。

正确做法：
- 只在“该系列当日已有真实任务实例”时，屏蔽父任务投影。

### 2. 不要破坏未来无实例日期的投影展示
当前日历页允许未来日期在无真实实例时仍显示周期任务存在感。

修复后应保留：
- 如果该日期没有真实实例，但父任务按重复规则命中，仍可显示一次

### 3. 分类筛选应在去重后还是前执行
推荐：
- 先按日期构造候选
- 再按系列去重
- 最后按 `currentCategory` 过滤

原因：
- 去重逻辑应在完整系列视角上运行，避免分类先过滤后误选错对象。
- 但如果系列内所有项分类本来一致，前后差异较小；仍推荐后过滤，逻辑更稳。

### 4. 状态切换命中的对象必须与展示对象一致
修复后：
- 列表里点完成/取消完成，应该操作当前展示出来的那一条真实任务或投影任务
- 尤其要确认：当列表显示的是当天实例时，不会误把父任务拿去切状态

## 建议实现步骤
1. 在 `calendar.js` 中新增“选中日期展示任务构建”辅助方法。
2. 将 `loadTasksForSelectedDate()` 改为使用该方法得到去重后的 `selectedDateTasks`。
3. 在去重中引入“真实当天任务优先，父任务投影补位”的规则。
4. 增加脏数据兜底选择策略，确保同系列同日最多一条。
5. 保持后续 `isOverdue`、`priorityColor`、分类名等派生逻辑不变。
6. 回归检查完成/取消完成后的本地刷新与整页 reload 场景。

## 验证清单
### 核心功能验证
1. 某周期系列父任务已完成、当天存在未完成实例时：
- 日历下方只显示 1 条
- 且显示在“进行中”
- 不再同时在“已完成”出现同名重复项

2. 选中父任务原始日期，且该父任务已完成时：
- 仍应显示 1 条已完成周期任务
- 不应被错误隐藏

3. 未来某个重复日期还没有真实实例时：
- 日历下方仍可显示 1 条周期任务投影
- 不应完全空白

4. 同一系列同一日期如果存在真实实例：
- 展示对象优先是真实实例
- 点完成/取消完成操作作用于该实例

### 状态分组验证
1. 同一周期系列不会同时出现在“进行中”和“已完成”
2. 同一周期系列不会同时出现在“进行中”和“已过期”
3. 非周期任务展示不受影响

### 回归验证
1. 日历徽章 `periodicCountMap` 数量不受影响
2. 点击灰色跨月日期后，去重逻辑仍正确
3. 分类筛选下，周期任务不再重复显示
4. 完成周期任务后触发 `loadTasks()` 的场景，刷新后仍无重复

## 最终建议
本问题应作为日历页的“展示层系列去重”来修，不建议动后端。

最小正确方案是：
- 在 `loadTasksForSelectedDate()` 中增加按系列去重
- 规则为“真实当天任务优先，父任务投影补位”

这样可以直接消除“进行中 + 已完成重复显示”的核心问题，同时不破坏当前周期规则与未来日期展示能力。
