# 共享清单邀请成员功能设计

## 1. 功能概述

重新设计共享清单的成员邀请功能，提供更友好的邀请体验，支持多种邀请方式，并完善邀请流程管理。

## 2. 核心功能模块

### 2.1 邀请方式

| 邀请方式 | 描述 | 适用场景 |
|---------|------|---------|
| 微信好友邀请 | 调用微信好友选择器，直接发送邀请卡片 | 邀请微信好友 |
| 生成邀请链接 | 生成可分享的邀请链接/二维码 | 邀请非微信好友或群聊 |
| 搜索用户ID | 输入用户ID或昵称搜索并邀请 | 知道对方ID时 |
| 最近协作成员 | 显示最近协作过的成员列表快速选择 | 重复邀请相同成员 |

### 2.2 邀请流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  选择邀请方式  │ -> │  选择被邀请人  │ -> │  设置权限角色  │ -> │  发送邀请    │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
                                                          │
                                                          v
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  成员加入清单  │ <- │  接受邀请    │ <- │  被邀请人收到  │ <- │  等待接受    │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

## 3. 数据库设计扩展

### 3.1 新增集合：list_invites（清单邀请记录）

| 字段名 | 数据类型 | 描述 | 索引 |
|--------|---------|------|------|
| _id | String | 邀请记录唯一标识 | 主键 |
| listId | String | 清单ID | 普通索引 |
| inviterId | String | 邀请人ID | 普通索引 |
| inviteeId | String | 被邀请人ID（可选，链接邀请时为空） | 普通索引 |
| inviteeInfo | Object | 被邀请人信息（微信邀请时存储） | 无 |
| role | Number | 邀请角色（2-编辑者，3-查看者） | 无 |
| inviteType | String | 邀请类型（wechat/link/search） | 无 |
| inviteCode | String | 邀请码（链接邀请时生成） | 唯一索引 |
| status | Number | 邀请状态（0-待接受，1-已接受，2-已拒绝，3-已过期） | 普通索引 |
| expireAt | Timestamp | 过期时间 | 普通索引 |
| createdAt | Timestamp | 创建时间 | 无 |
| updatedAt | Timestamp | 更新时间 | 无 |

### 3.2 扩展 list_members 集合

新增字段：
| 字段名 | 数据类型 | 描述 |
|--------|---------|------|
| joinType | String | 加入方式（invite-被邀请，link-链接加入，create-创建者） |
| inviteId | String | 关联的邀请记录ID |

## 7. 业务规则

### 7.1 邀请有效期

- 微信邀请：7天有效期
- 邀请链接：可设置（1天/7天/30天/永久）
- 过期后邀请自动失效，可重新发起

### 7.2 邀请次数限制

- 同一用户同一清单最多3次待处理邀请
- 同一邀请码最多使用10次（链接邀请）
- 防止恶意刷邀请

### 7.3 成员数量限制

- 免费版：最多10个成员/清单
- 付费版：最多100个成员/清单
- 达到上限时提示升级或移除现有成员

### 7.4 重复邀请处理

- 已是成员的用户：提示"该用户已是成员"
- 已有待处理邀请：提示"已发送邀请，请勿重复发送"
- 被拒绝过的用户：可重新邀请

## 8. 通知机制

### 8.1 邀请发送通知

被邀请人收到：
- 微信服务通知（小程序）
- 应用内消息中心通知
- 可选：短信/邮件通知

### 8.2 邀请状态变更通知

邀请人收到：
- 被邀请人接受邀请
- 被邀请人拒绝邀请
- 邀请即将过期提醒（24小时前）

### 8.3 系统通知内容模板

```
【邀请通知】
标题：张三邀请你加入"工作任务"清单
内容：你收到一个待办清单协作邀请，点击查看详情
操作：接受 / 拒绝 / 查看详情
```

## 9. 安全考虑

### 9.1 邀请码安全

- 邀请码使用随机字符串（16位）
- 包含时间戳和校验位，防止暴力破解
- 单次使用或限时使用

### 9.2 链接安全

- 邀请链接包含签名验证
- 防止链接篡改和伪造
- 链接过期后自动失效

### 9.3 权限验证

- 每次操作验证用户权限
- 防止越权访问和操作
- 敏感操作二次确认

## 11. 接口设计

### 11.1 核心API

```javascript
// 发起微信邀请
wx.cloud.callFunction({
  name: 'inviteMemberByWechat',
  data: {
    listId: 'list_xxx',
    inviteeInfo: { openid: 'xxx', nickname: 'xxx' },
    role: 2
  }
})

// 生成邀请链接
wx.cloud.callFunction({
  name: 'generateInviteLink',
  data: {
    listId: 'list_xxx',
    role: 2,
    expireDays: 7,
    needApproval: false
  }
})

// 搜索用户
wx.cloud.callFunction({
  name: 'searchUser',
  data: {
    keyword: '张三'
  }
})

// 接受邀请
wx.cloud.callFunction({
  name: 'acceptInvite',
  data: {
    inviteId: 'invite_xxx',
    inviteCode: 'abc123' // 链接邀请时使用
  }
})

// 拒绝邀请
wx.cloud.callFunction({
  name: 'rejectInvite',
  data: {
    inviteId: 'invite_xxx'
  }
})

// 获取邀请列表
wx.cloud.callFunction({
  name: 'getInviteList',
  data: {
    listId: 'list_xxx',
    status: 0 // 0-待处理
  }
})

// 取消邀请
wx.cloud.callFunction({
  name: 'cancelInvite',
  data: {
    inviteId: 'invite_xxx'
  }
})
```
